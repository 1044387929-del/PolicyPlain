## 2. 系统分层总览

逻辑上仍为 **浏览器 → FastAPI →（PostgreSQL / Redis / 外部 HTTP / 大模型）**；在应用内部再拆 **接入层、领域编排、基础设施**。

```text
                    ┌─────────────────────────────────────────┐
                    │           policy-frontend (Vue)          │
  表现 + 交互        │  router → pages → components            │
                    │  stores / composables / apis              │
                    └──────────────────┬────────────────────────┘
                                       │ HTTPS JSON / SSE
                    ┌──────────────────▼────────────────────────┐
                    │           policy-backend (FastAPI)       │
  接入层             │  routers/     HTTP、鉴权、StreamingResponse │
                    ├──────────────────┬────────────────────────┤
  编排 + 业务        │  routers 内联   SSE 生成器、URL+解读流水线   │
  （部分在 router）   │  services/    LLM、网页抓取、流式 partial  │
                    ├──────────────────┼────────────────────────┤
  数据与外部 IO      │  repository/  按聚合根的异步 CRUD           │
                    │  models/      ORM 实体 + Session + JWT 封装 │
                    │  core/        Redis 封装、邮件、OCR        │
                    ├──────────────────┼────────────────────────┤
  契约与配置         │  schemas/     Pydantic 请求体、响应体       │
                    │  settings/    环境变量集中读取               │
                    │  dependencies/ FastAPI Depends 工厂        │
                    └──────────────────┬────────────────────────┘
                                       │
              ┌────────────────────────┼────────────────────────┐
              ▼                        ▼                        ▼
        PostgreSQL                  Redis                 外网 HTTP / LLM API
```

**依赖方向（约定）**：`routers` → `services` / `repository` / `core` / `schemas`；`repository` → `models`；**不**让 `models` 或 `repository` 依赖 `routers`。配置统一由 `settings` 读取，避免在深层模块硬编码环境变量。

## 3. policy-backend 代码分层

### 3.1 分层与目录对照

| 层级 | 目录 / 文件 | 职责 |
| --- | --- | --- |
| **应用入口** | `main.py` | 创建 FastAPI 应用、`lifespan`（**Redis 客户端**创建/关闭、`set_redis_client`）、CORS、挂载 **`/api/v1`** 下各 router |
| **路由（接入层）** | `routers/health.py` | 健康检查 |
| | `routers/user_router.py` | 注册发码、注册、登录、me、logout；依赖 **`PolicyCache`**、**`UserRepo`**、**`BackgroundTasks`**（邮件） |
| | `routers/policy.py` | 白话解读 SSE、网址兼容接口、OCR、历史列表/详情、**追问 SSE**；内联 **`_stream_mixed_explain`**、**`_stream_explain_and_save`**、**`_stream_follow_up_answer`** 等**长流程编排** |
| **领域服务** | `services/explain_llm.py` | **System/User 提示词**、**流式/非流式** OpenAI 兼容调用、mock 分片、**`normalize_to_schema`** |
| | `services/url_fetch.py` | 公网 URL 校验、抓取 HTML、转纯文本 |
| | `services/streaming_partial.py` | 流式累积 JSON 的 **lenient 解析**，供 SSE **`partial`** |
| **数据访问** | `repository/user_repo.py` | 用户按邮箱/ID 查询、创建用户 |
| | `repository/policy_repo.py` | 解读记录、追问记录的增删查（按用户隔离） |
| **持久化模型** | `models/user.py` | SQLAlchemy 实体：`UserModel`、`PolicyExplanationModel`、`PolicyFollowupModel` 等 |
| | `models/__init__.py` | **`engine`**、**`AsyncSessionFactory`**、**`get_session()`**、**`Base`**、**`auth_handler`** 实例 |
| **基础设施** | `core/cache.py` | **`PolicyCache`**：`redis.asyncio` 上的注册验证码键 **`policy-plain:register:`** |
| | `core/auth.py` | JWT 签发与 **`auth_access_dependency`** |
| | `core/mail.py` | SMTP 发信（注册验证码邮件） |
| | `core/ocr.py` | **PaddleOCR** 云端调用封装 |
| **请求/响应契约** | `schemas/*.py` | Pydantic：登录/注册、政策 explain、列表项、追问 body 等 |
| **依赖注入** | `dependencies/__init__.py` | **`get_session_instance`**、**`get_cache_instance`**、**`get_current_user`**（组合 JWT + `UserRepo`） |
| **异步任务（轻量）** | `tasks/__init__.py` | 注册邮件发送任务（由 router 挂到 `BackgroundTasks`） |
| **配置** | `settings/__init__.py` | **`pydantic-settings`** 聚合环境变量（数据库、Redis、邮件、LLM、OCR、字数上限等） |
| **数据库迁移** | `alembic/` | 版本化 DDL；**应用启动不自动建表** |

### 3.2 架构所在（按阅读顺序找代码）

1. **HTTP 路由表**：`policy-backend/main.py` 的 `include_router`。  
2. **鉴权从哪注入**：`dependencies/__init__.py` → `get_current_user` / JWT。  
3. **一次解读的完整后端链路**：`routers/policy.py`（编排）→ `services/url_fetch.py`（若有 URL）→ `services/explain_llm.py`（synopsis + 主解读流）→ `services/streaming_partial.py`（partial）→ `repository/policy_repo.py`（落库）。  
4. **提示词与模型调用**：仅 **`services/explain_llm.py`**（与路由解耦，便于单测与替换供应商）。  
5. **表结构**：`models/user.py` + **`alembic/versions/`**。

### 3.3 编排落点说明（与「严格三层」的差异）

本项目体量较小，**部分业务流程写在 `routers/policy.py` 的异步生成器里**（例如 `_stream_mixed_explain`：抓取 → synopsis → 再调 `_stream_explain_and_save`）。这样 SSE 与 `yield sse_event(...)` 集中在同一文件，减少跳转成本。

若后续膨胀，可将这些生成器迁至 **`services/explain_orchestrator.py`**（或类似命名），router 只负责 `Depends` 与 `StreamingResponse`，分层会更「教科书」，但当前结构在笔试规模下是刻意取舍。

### 3.4 典型调用链（简图）

**注册发码**

```text
user_router.send_register_code
  → PolicyCache.set_register_info (core/cache + Redis)
  → UserRepo.get_by_email (repository)
  → BackgroundTasks → tasks.send_register_code_email_task (core/mail)
```

**白话解读（含 URL）**

```text
policy.explain → _stream_mixed_explain
  → [可选] url_fetch → extract_synopsis_from_web_plain (explain_llm)
  → _stream_explain_and_save → llm_token_stream (explain_llm)
       → try_normalize_streaming_explain (streaming_partial) → SSE partial
  → json 校验 + normalize_to_schema → PolicyRepo.create → SSE done
```

## 4. policy-frontend 代码分层

前端采用 **Vue 3 SPA**，按「路由 → 页面 → 组件 / 状态 / API」组织，**无**单独 `views` 与 `pages` 双目录，页面均在 **`src/pages/`**。

| 层级 | 路径 | 职责 |
| --- | --- | --- |
| **入口** | `main.ts`、`App.vue` | 挂载应用、根布局 |
| **路由** | `router/index.ts` | 路由表、**`beforeEach`**（`requiresAuth` / `public`、登录重定向） |
| **布局** | `components/AppLayout.vue`、`AuthLayout.vue` | 已登录壳子（顶栏、出口）与登录注册布局 |
| **页面（特性聚合）** | `pages/explain/index.vue` | 解读主流程：表单、URL、OCR、**SSE 消费**、侧栏/抽屉 |
| | `pages/login`、`pages/register` | 认证表单 |
| | `pages/history/index.vue`、`detail.vue` | 历史列表与详情 |
| **可复用展示** | `components/ExplainResultElder.vue` | 结构化结果适老化展示 |
| | `components/FollowUpChat.vue` | 追问与 SSE |
| | `components/DisplayComfortBar.vue` | 全局字号适老 |
| **API 客户端** | `apis/request.ts` | Axios 实例、Base URL、**Authorization** 拦截器 |
| | `apis/user_api.ts`、`apis/policy_api.ts` | 按领域封装的请求与 **SSE 解析**（如 `explainPolicyStream`） |
| **状态** | `stores/user.ts` | Token、用户信息、是否登录 |
| | `stores/displayComfort.ts` | 字号档位、`localStorage` |
| **组合逻辑** | `composables/usePolicySpeak.ts` | 朗读等与结果相关的可复用逻辑 |
| **常量** | `constants/elderImagery.ts` | 适老主题图等资源路径 |

**依赖方向**：`pages` → `apis` / `stores` / `components` / `composables`；**避免** `apis` 引用 `pages`（保持 HTTP 层可复用）。

### 4.1 架构所在（前端）

- **路由与鉴权门槛**：`src/router/index.ts`。  
- **所有后端 JSON/SSE 入口**：`src/apis/`。  
- **解读页最复杂状态机**：`src/pages/explain/index.vue`（与概要设计中的主界面一致）。
