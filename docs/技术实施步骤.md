# PolicyPlain 技术实施步骤（笔试两天节奏）

依据：`项目需求分析.md`、`项目概要设计.md`（以代码落地为准时可微调字段）、参考工程 `hr-backend`、`hr-frontend-src` 的技术风格。

## 1. 技术选型（与参考工程对齐）


| 层级       | 选型                                                                                     | 说明                                                      |
| -------- | -------------------------------------------------------------------------------------- | ------------------------------------------------------- |
| 后端       | FastAPI + Uvicorn                                                                      | 与 hr-backend 一致                                         |
| ORM / DB | SQLAlchemy 2.x 异步 + **PostgreSQL**（`asyncpg`）                                          | 本地先 `CREATE DATABASE`；连接见 `policy-backend/.env.example` |
| 密码       | `pwdlib`（argon2）                                                                       | 与 hr-backend `UserModel` 思路一致                           |
| 鉴权       | PyJWT + `HTTPBearer`，`iss` 存 user id                                                   | 与 hr-backend `core/auth.py` 模式一致                        |
| 前端       | Vue 3 + Vite + TypeScript + **Pinia** + **Axios** + **Element Plus** + **Tailwind v4** | 与 hr-frontend-src 一致                                    |
| LLM      | `openai` 官方异步 SDK（兼容 OpenAI 协议 Base URL）                                               | 未配置 API Key 时走 **mock 结构化结果**，保证无密钥也能演示                 |

## 3. 执行顺序（建议按天切分）

### Day 0.5：环境与骨架

1. 创建 `policy-backend`：`settings`、`models`（`users`、`policy_explanations`）、`dependencies`、`core/auth`（从 hr 精简）、`main.py` 挂 CORS、`GET /api/v1/health`。
2. **Alembic** 管理表结构（与 `hr-backend` 一致）：`alembic/` + `scripts/init_db.*`；启动前 `alembic upgrade head`，不在应用内 `create_all`。
3. 创建 `policy-frontend`：`package.json`、`vite`/`tailwind` 配置、`request` 拦截器、`pinia` user store（token 存 localStorage，与 hr 一致）。

### Day 1 上午：账号闭环

1. 实现 `POST /api/v1/auth/register`、`POST /api/v1/auth/login`、`GET /api/v1/users/me`。
2. 前端 `/login`、`/register`、路由守卫；联调拿到 token。

### Day 1 下午：核心业务

1. 实现 `POST /api/v1/policy/explain`：校验长度 → 调 LLM 或 mock → 写库 → 返回 `record_id` + 结构化字段。
2. 前端 `/explain`：免责声明、主题、粘贴、结果卡片。

### Day 2 上午：历史与 polish

1. `GET /api/v1/policy/explanations`、`GET /api/v1/policy/explanations/{id}`，强制 `user_id` 过滤防越权。
2. 前端 `/history`、`/history/:id`；适老化字号/间距。

### Day 2 下午：交卷材料

1. 根目录 `README.md`：复制 `.env.example`、安装命令、`uvicorn`/`npm run dev`、默认账号演示路径。
2. 自测：注册 → 解读 2 段样例 → 历史回看；检查未登录 401。

## 4. 刻意不做（控制范围）

- Redis、邮件、钉钉、定时任务（hr-backend 的 lifespan 重量级能力一律不要）。
- 邀请制注册（hr 的 invite）；本项笔试用「**邮箱** + 密码 + 确认密码」开放注册即可。
